      Frequently Asked Questions

      Things people usually ask me on LinkedIn. Saving you the DM wait.

      Start with Linux (not Windows). Spend 2–3 weeks getting comfortable with the terminal, file permissions, and Bash scripting. Then learn Git properly — not just git add/commit/push, but branching, merging, and rebasing. After that: Docker, then Kubernetes. Check my Resources page — I've listed everything I personally used in the exact order I'd recommend.
      No — but you need proof of skills. Recruiters want to see GitHub repos with real projects: a CI/CD pipeline, a Kubernetes deployment, an IaC setup on AWS. Build public projects, write a README explaining what problem each project solves, and put the GitHub link on your resume. That matters more than the degree in most DevOps hiring.
      AIOps (AI for IT Operations) uses machine learning and LLMs to automate what DevOps engineers currently do manually — anomaly detection, root cause analysis, and incident remediation. My research builds a multi-agent LLM system that watches a Kubernetes cluster via Prometheus + Loki + Jaeger and autonomously fixes issues. It's DevOps, but the on-call engineer is an AI. Check the AIOps page for details.
      Absolutely — I'm open to collaborations on open-source DevOps tooling, AIOps research, or cloud-native projects.       Start with the Linux Foundation's free LFS101 (Introduction to Linux) — it's free and covers exactly what you need. Then AWS Cloud Practitioner for cloud fundamentals. Once you're comfortable with Kubernetes, go for CKA (Certified Kubernetes Administrator) — it's the most respected hands-on cert in DevOps. See my full list on the Resources page.
